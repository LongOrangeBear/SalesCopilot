## Этап 1 -- Инфраструктура и сервер

**Цель:** тестовый звонок проходит через наш сервер, два аудиопотока видны в логах.

- [x] VPS -- получить доступ (FastVPS, 5.45.112.38)
- [x] Подготовка сервера (swap, обновления)
- [x] Установка Asterisk 20 LTS
- [x] Настройка PJSIP (transport UDP+TCP, NAT traversal)
- [x] Тестовые SIP-аккаунты (manager/100, client/200)
- [x] Dialplan с MixMonitor (запись звонков)
- [x] Firewall (UFW: SIP 5060, RTP 10000-20000)
- [x] Тест: Zoiper подключен, эхо-тест (ext 600) работает
- [x] Документация деплоя (asterisk_deploy.md)
- [ ] SIP-провайдер (Mango Office) -- получить аккаунт и SIP-транк
- [ ] Настроить SIP-транк в Asterisk (входящий маршрут от провайдера)
- [ ] Настроить failover в Mango Office (если Asterisk недоступен -> звонок напрямую менеджеру)
- [ ] Тест: реальный входящий звонок на номер провайдера -> проходит через Asterisk -> менеджер слышит

## Этап 2 -- Распознавание речи + CRM

**Цель:** во время звонка в браузере появляется транскрипт + карточка клиента из CRM.

### 2.1 Бэкенд (Python/FastAPI)

- [x] Структура проекта (FastAPI, WebSocket, конфиг)
- [ ] Интеграция с Asterisk: получение аудиопотоков (ARI или AudioSocket)
- [x] Объект CallSession (in-memory: аудио, WebSocket, контекст клиента, реплики, таймстампы)

### 2.2 Админ-дашборд (отладка и мониторинг)

Веб-интерфейс для разработки и отладки -- видеть всё, что происходит внутри системы.

- [x] **Монитор активных звонков** -- список текущих звонков, кто звонит, кто говорит прямо сейчас, длительность
- [x] **Live-транскрипт** -- текст диалога в реальном времени с пометками "Клиент" / "Менеджер"
- [x] **Лог ИИ-запросов** -- отдельная панель: что отправили в LLM (промпт, контекст) и что получили (подсказка, время ответа)
- [x] **Тайминги пайплайна** -- задержки каждого этапа: передача аудио, STT, LLM, доставка подсказки
- [x] **Архив звонков** -- список завершённых звонков с записями, расшифровками, сводками
- [x] **Детали звонка** -- запись (аудиоплеер), полный транскрипт, все подсказки ИИ, тайминги, данные клиента из CRM
- [x] **Системные метрики** -- загрузка CPU/RAM, статус Asterisk, статус STT/LLM сервисов

### 2.3 Распознавание речи (Yandex SpeechKit)

- [x] Аккаунт Yandex Cloud + API-ключ SpeechKit
- [x] Streaming STT: два параллельных потока (клиент + менеджер)
- [x] Транскрипт с пометками "Клиент" / "Менеджер"

### 2.4 Интеграция с CRM (Bitrix24)

- [ ] Тестовый портал Bitrix24 + доступ к API
- [ ] CRM-адаптер: поиск контакта по номеру телефона при входящем звонке
- [ ] Подгрузка данных сделки (имя, компания, стадия, бюджет)
- [ ] Запись саммари и задач после звонка

### 2.5 Контрольная точка
