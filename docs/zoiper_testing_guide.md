# Тестирование SalesCopilot через Zoiper

Пошаговая инструкция по настройке Zoiper и проверке работы телефонии + дашборда.

---

| Платформа | Ссылка |

## 3. Настройка аккаунта Manager

### 3.1. Создать аккаунт

1. Открыть Zoiper -> **Settings** -> **Accounts** -> **Add account**
2. Выбрать **SIP account** (или "I already have a SIP account")
3. Заполнить поля:

| Поле | Значение |
|---|---|
| **Account name** | Manager (любое имя) |
| **Domain / Host** | `5.45.112.38` |
| **Username** | `manager` |
| **Password** | `Manager123!` |
| **Auth username** | `manager` |

4. Нажать **Register** / **Save**

### 3.2. Настройки сети

1. Перейти в **Settings** -> **Accounts** -> **Manager** -> **Network settings**
2. **Transport**: выбрать **UDP** (или TCP если через VPN)
3. **STUN**: **отключить** (Asterisk сам решает NAT)
   - Advanced -> STUN -> None / Disabled

### 3.3. Проверка статуса

- Статус аккаунта должен быть **"Registered"** или **"Online"** (зеленая иконка)
- Если "Registration failed" -- проверить логин/пароль и доступность порта 5060

## 5. Тестирование звонков

### Тест 1: Эхо-тест (проверка аудио)

1. С **любого** аккаунта набрать **600**
2. После соединения говорить в микрофон
3. Если слышишь свой голос с задержкой -- аудио работает корректно
4. Повесить трубку

> Если звук не идёт -- проверить настройки микрофона в Zoiper и отключение STUN.

### Тест 2: Звонок Manager -> Client

1. С аккаунта **Manager** набрать **200**
2. На устройстве **Client** должен прийти входящий звонок
3. Принять звонок на Client
4. Проверить двустороннюю связь (говорить и слушать с обеих сторон)
5. Повесить трубку

### Тест 3: Звонок Client -> Manager

1. С аккаунта **Client** набрать **100**
2. На устройстве **Manager** должен прийти входящий звонок
3. Принять и проверить связь

## 6. Проверка на дашборде SalesCopilot

Дашборд доступен по адресу: **http://5.45.112.38:3211**

### 6.1. Список звонков

После завершения тестовых звонков:

1. Открыть дашборд в браузере
2. Перейти в раздел **"Звонки"** (Calls)
3. В списке должны появиться записи о совершённых звонках с информацией:
   - Дата и время
   - Кто звонил (CallerID)
   - Кому звонили (номер назначения)
   - Длительность
   - Статус (отвечен / не отвечен)

### 6.2. Записи звонков

- Записи хранятся на сервере в `/var/spool/asterisk/monitor/`
- Формат: WAV
- Имя файла: `YYYYMMDD-HHMMSS-{caller}-to-{destination}.wav`
- На дашборде можно прослушать запись прямо в браузере (если функция подключена)

### 6.3. Аналитика

- После обработки записи SalesCopilot через AI (транскрипция + анализ):
  - Появится текст разговора
  - Оценка качества звонка
  - Рекомендации менеджеру

> На этапе тестирования аналитика может быть ещё не подключена -- это нормально. Главное убедиться, что звонки проходят и записываются.

## 7. Устранение проблем

| Проблема | Решение |
|---|---|
| Registration failed | Проверить username/password. Убедиться что порт 5060 открыт (UDP) |
| Нет звука | Отключить STUN в Zoiper. Проверить что порты 10000-20000/UDP открыты на сервере |
| Звонок не проходит (404) | Убедиться что набираете номер (**100** / **200**), а не имя. Проверить что второй аккаунт зарегистрирован |
| Односторонний звук | Отключить STUN. Проверить `direct_media=no` в pjsip.conf |
| "Temporarily Unavailable" | Второй абонент не зарегистрирован в Zoiper (приложение закрыто или офлайн) |

### Проверка со стороны сервера

```bash
# Подключиться к серверу
ssh root@5.45.112.38

# Проверить зарегистрированные устройства
asterisk -rx 'pjsip show contacts'

# Посмотреть активные звонки
asterisk -rx 'core show channels'

# Логи в реальном времени
tail -f /var/log/asterisk/messages.log

# Проверить записи звонков
ls -la /var/spool/asterisk/monitor/
```
